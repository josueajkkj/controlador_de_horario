## Día 1 — Entender el Sistema

### ¿Qué es Vibe Coding?

Vibe Coding es una forma de desarrollo donde el programador utiliza herramientas de inteligencia artificial para acelerar la creación de software.  
En lugar de escribir todo el código manualmente desde cero, se trabaja con prompts, planificación y asistencia de modelos IA para generar, estructurar y mejorar partes del proyecto.

Su objetivo es aumentar la productividad, reducir tiempo de desarrollo y enfocarse más en la lógica, el diseño y la toma de decisiones que en tareas repetitivas.

---

### Diferencia entre entorno web y entorno local

#### Entorno web
El entorno web es cuando usamos una herramienta directamente desde el navegador, sin necesidad de instalarla en la computadora.

**Ventajas:**
- Acceso rápido
- No requiere instalación
- Fácil de usar desde cualquier equipo con internet

**Desventajas:**
- Depende de conexión a internet
- Puede tener limitaciones frente a un entorno local
- Menor control sobre archivos y configuración

#### Entorno local
El entorno local es cuando instalamos las herramientas directamente en nuestra computadora.

**Ventajas:**
- Mayor control del proyecto
- Mejor integración con archivos, carpetas y terminal
- Más flexibilidad para desarrollo real

**Desventajas:**
- Requiere instalación y configuración
- Puede ser más complejo al inicio

**Conclusión:**  
El entorno web es ideal para empezar rápido, mientras que el entorno local es mejor para proyectos más completos y profesionales.

---

### ¿Qué es modo Planning?

El modo **Planning** es una forma de trabajo donde la inteligencia artificial se enfoca primero en **pensar, estructurar y planificar** antes de generar código.

En este modo, la IA:
- Analiza el problema
- Divide el trabajo en pasos
- Propone arquitectura o estructura
- Reduce errores antes de programar

**Es útil cuando:**
- El proyecto es grande
- Queremos una solución ordenada
- Necesitamos entender la lógica antes de ejecutar

---

### ¿Qué es modo Fast?

El modo **Fast** es una forma de trabajo donde la inteligencia artificial responde de manera **rápida y directa**, priorizando velocidad sobre análisis profundo.

En este modo, la IA:
- Genera resultados rápidos
- Es útil para tareas pequeñas o repetitivas
- Ayuda en pruebas, snippets o cambios simples

**Es útil cuando:**
- Queremos avanzar rápido
- El problema es sencillo
- Necesitamos una respuesta inmediata

---

### ¿Qué modelo usarías para generar una app completa y por qué?

Para generar una app completa, usaría un modelo enfocado en **razonamiento, planificación y generación de código de buena calidad**, porque una aplicación necesita:

- Buena estructura
- Separación de responsabilidades
- Organización de archivos
- Menos errores
- Capacidad de mantener el proyecto escalable

Elegiría un modelo que funcione mejor en tareas complejas y que pueda ayudar en:

- Arquitectura del proyecto
- Generación de componentes
- Lógica de negocio
- Depuración de errores
- Documentación

**¿Por qué?**  
Porque una app completa no solo necesita código rápido, sino también **orden, claridad y buena planificación**.  
Por eso, para una aplicación grande conviene priorizar un modelo más preciso y estructurado, similar a trabajar en modo **Planning**.

---

## Conclusión

En este primer día aprendí que Vibe Coding combina programación con inteligencia artificial para desarrollar más rápido y de forma más estratégica.  
También entendí la diferencia entre trabajar en entorno web y entorno local, y cuándo conviene usar modo Planning o modo Fast según el tipo de tarea.  
Finalmente, comprendí que para construir una app completa es mejor usar un modelo que piense bien la estructura y no solo que responda rápido.
